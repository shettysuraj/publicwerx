# Contributing to PublicWerx

## Before You Start

Read the [constitution](CONSTITUTION.md). All of it. If you do not agree with the tenets, this is not the right project for you, and that is fine.

## What You Can Contribute

- **Constitution improvements.** Clarifications, better wording, edge cases. The tenets themselves are stable, but the surrounding text can always be sharper.
- **Site improvements.** The publicwerx.org site is static HTML. Bug fixes, accessibility improvements, and design refinements are welcome.
- **New charter adopters.** If your project adopts the constitution, open a PR to add it to `CHARTER-ADOPTERS.md`.
- **Sub-project work.** PublicWerx hosts sub-projects (like the Android hardware wallet payment app). Contributions to those repos follow their own contributing guides, but the constitutional alignment applies to all of them.

## How to Submit a PR

1. Fork the repo.
2. Make your changes on a branch.
3. Open a pull request with a clear description of what you changed and why.
4. Keep it focused. One concern per PR.

## Standards

- No corporate language. Write like a human.
- No unnecessary complexity. Stay Lean applies to contributions too.
- If your change affects users in any way, document it. Radical Transparency is not optional.
- No silent changes to anything that affects data practices or privacy. Ever.

## Corporate Contributors

If you are contributing on behalf of a company, your organization must explicitly align with the constitution. We are not interested in contributions that come with strings attached, or from organizations whose business model contradicts the tenets.

## Decision Making

All decisions that affect the project are made transparently and documented publicly. There are no private channels, no backroom agreements, no silent policy changes. If it matters, it is written down where everyone can see it.

## License

By contributing, you agree that your contributions are licensed under GPL-3.0, consistent with the rest of the project.
