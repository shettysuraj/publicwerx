## How to Get Listed

1. Read and adopt the [constitution](CONSTITUTION.md).
2. Add a `CONSTITUTION.md` to your own repo.
3. Include a succession plan (see [SUCCESSION.md](SUCCESSION.md) for a template).
4. Open a PR adding your project to this file.

## Accountability Process

Any member of the community can challenge any listed project -- including PublicWerx itself -- for violating the tenets. No project is above the constitution it adopted.

### How it works

1. **Raise the issue.** Open a GitHub issue on the [publicwerx repo](https://github.com/shettysuraj/publicwerx) citing the specific tenet and the specific violation. Be concrete.
2. **The project responds.** The accused project has 30 days to respond publicly in the issue thread. They can explain, dispute, or commit to fixing the violation.
3. **Resolution.** If the violation is acknowledged and fixed, the issue is closed. If the project disputes the claim, the community discusses it openly in the thread.
4. **Delisting.** If a violation is clear, unresolved, and the project has refused to address it, a PR is opened to move the project from "Active Projects" to "Delisted Projects" with a public record of why.

Delisting is a last resort, not a first response. The goal is to help projects stay in compliance, not to punish them. But a list with no accountability is just marketing.
